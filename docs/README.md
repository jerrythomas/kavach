# Documentation

Project documentation organized by purpose.

## Contents

| Folder | Purpose |
|--------|---------|
| [requirements/](requirements/) | Feature requirements — the "what" |
| [design/](design/) | Module design documents — the "how" |
| [examples/](examples/) | Usage examples and getting started guides |

## Conventions

- **Numbered prefixes** (`01-xxx.md`, `02-xxx.md`) for ordered documents within each folder
- **README.md** in each subfolder for navigation and context
- Requirements are reference-only (not updated after initial agreement)
- Design docs are living documents (updated as implementation evolves)
