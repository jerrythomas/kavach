# 05 — UI Components

## What

The UI package (`@kavach/ui`) provides pre-built Svelte components for authentication flows. It includes:

- **Auth form components** — buttons, inputs, and handlers for sign-in/sign-up forms
- **Provider grouping** — organize auth providers visually (social, password, magic link)
- **Login cards** — cached login entries showing previous sign-ins with avatars and provider badges
- **Auth page layout** — smart orchestrator that shows cached logins or falls back to provider list
- **Error/response display** — components for auth feedback messages

## Why

Every app with authentication needs a login page. Building one from scratch means handling form state, error display, provider buttons, and loading states. The UI package provides:

1. **Ready-made components** — drop in `<AuthPage>` and get a working login page
2. **Consistent UX** — error handling, loading states, and success feedback work the same everywhere
3. **Cached logins** — returning users see their previous accounts for one-click re-authentication
4. **Adapter-agnostic** — components work with any adapter; they call `kavach.signIn()` not provider methods

## Scope

### In scope
- `AuthPage` — full-page auth layout with cached login cards and provider list
- `AuthHandler` — form submission handler that routes to kavach signIn/signUp
- `AuthButton` — styled button for auth actions
- `AuthProvider` — single provider entry (icon + label)
- `AuthGroup` — group providers with a section header
- `AuthError` — display error messages from auth operations
- `AuthResponse` — display success/info/warning feedback
- `LoginCard` — single cached login with avatar, name, provider badge, remove action
- `LoginCardList` — list of LoginCard components
- Internal: `AuthPassword`, `EmailInput`, `InputField`, `Button`

### Out of scope
- Styling framework (consumers bring their own CSS/Tailwind/UnoCSS)
- Route protection UI (handled by app code)
- Provider SDK configuration UI (handled by consumers)

## Component Hierarchy

```
AuthPage
  ├── LoginCardList (if cached logins exist)
  │     └── LoginCard (per cached entry)
  └── AuthHandler (sign-in/sign-up form)
        ├── AuthGroup
        │     └── AuthProvider (per provider)
        ├── EmailInput
        ├── AuthPassword
        ├── AuthButton
        ├── AuthError
        └── AuthResponse
```

## Dependencies

- `kavach` (`@kavach/auth`) — auth state, login cache
- `@rokkit/forms` — form utilities
- `ramda` — functional utilities
