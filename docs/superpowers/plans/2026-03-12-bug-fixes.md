### Task 2: Fix the three Response body locations

- [ ] **Fix `handleUnauthorizedAccess` — redirect branch (line ~116)**

  In `packages/auth/src/kavach.js`, find:

  ```js
  return new Response(
    {},
    {
      status: 303,
      headers: { location: event.url.origin + result.redirect }
    }
  )
  ```

  Replace with:

  ```js
  return new Response('', {
    status: 303,
    headers: { location: event.url.origin + result.redirect }
  })
  ```

- [ ] **Fix `handleUnauthorizedAccess` — error branch (line ~123)**

  Find:

  ```js
  return new Response({ error: HTTP_STATUS_MESSAGES[result.status] }, { status: result.status })
  ```

  Replace with:

  ```js
  return new Response(JSON.stringify({ error: HTTP_STATUS_MESSAGES[result.status] }), {
    status: result.status,
    headers: { 'Content-Type': 'application/json' }
  })
  ```

- [ ] **Fix `handleSessionSync` — response (line ~179)**

  Find:

  ```js
  return new Response({ session, error }, { status, headers })
  ```

  Replace with:

  ```js
  return new Response(JSON.stringify({ session, error }), {
    status,
    headers: { ...Object.fromEntries(headers), 'Content-Type': 'application/json' }
  })
  ```

  Note: `headers` in `handleSessionSync` is a cookie header object from `setCookieFromSession`. Check the return type of `setHeaderCookies` — if it returns a plain object (not a `Headers` instance), the spread above works. If it returns a `Headers` instance, use:

  ```js
  const responseHeaders = new Headers(headers)
  responseHeaders.set('Content-Type', 'application/json')
  return new Response(JSON.stringify({ session, error }), { status, headers: responseHeaders })
  ```

### Task 3: Add `/data/facts` routing rule

- [ ] **Open `sites/learn/kavach.config.js` and add the rule**

  Find the `rules` array and add two entries:

  ```js
  rules: [
    { path: '/', public: true },
    { path: '/docs', public: true },
    { path: '/auth', public: true },
    { path: '/demo', roles: '*' },
    { path: '/data/facts', roles: '*' }, // ← add
    { path: '/data/admin-stats', roles: ['admin'] } // ← add (for future demo use)
  ]
  ```

  Rationale: `/data` is configured as an endpoint route in the guardian. Without an explicit rule, authenticated users get 403. Adding `roles: '*'` allows any authenticated user to fetch space facts; `roles: ['admin']` restricts the admin-stats route.

### Task 4: Remove always-on debug log from hooks

- [ ] **Open `sites/learn/src/hooks.server.js`**

  Current content:

  ```js
  import { kavach } from '$kavach/auth'

      `[hook] ${event.url.pathname} → status=${result?.status} location=${result?.headers?.get('location')}`
    )

  }
  ```

  Replace with:

  ```js

  export const handle = async ({ event, resolve }) => {
    return kavach.handle({ event, resolve })
  }
  ```
